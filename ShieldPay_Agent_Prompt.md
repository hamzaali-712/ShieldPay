## DOCUMENT CONTEXT — READ THIS FIRST

You are building **ShieldPay**. Here is everything you need extracted from the PRD and TRD:

### What ShieldPay Is
A real-time, AI-driven financial safety guardrail for Gen-Z banking users in Pakistan. It embeds an AI inference layer into the digital banking workflow so students can detect scams, validate payments, and build financial literacy — all in one place.

### The 3 Core Features You Must Build

**Feature 1 — Scam Content Analyzer ("Scan Content" tab)**
- A large textarea where users paste suspicious SMS, WhatsApp messages, or URLs
- On submit, call the Anthropic Claude API (claude-sonnet-4-6) from the backend
- Return and display: classification (SAFE / SUSPICIOUS / DANGEROUS), confidence score (0-100), threat type, a list of red flags, a plain-English explanation, and a recommended action
- Color code the result card: green for SAFE (#00A86B), amber for SUSPICIOUS (#D97706), crimson for DANGEROUS (#DC2626)

**Feature 2 — P2P Transfer Validator ("Validate Transfer" tab)**
- A mock UBL-styled payment form with: Recipient Name, IBAN (format: PK + 2 digits + 4 bank code letters + 16 digits), Amount in PKR, and Transfer Note
- Validate IBAN format with regex on the frontend
- Send to backend, run AI behavioral analysis on the transfer note and amount patterns
- Return and display: IBAN validity, resolved bank name, risk level (LOW / MEDIUM / HIGH), risk score, behavioral flags, an AI summary, and a proceed recommendation (PROCEED / VERIFY_FIRST / DO_NOT_PROCEED)

**Feature 3 — Student Education Hub ("Learn & Earn" tab)**
- 5 interactive learning modules covering Pakistan-specific scam types:
  1. Fake Prize SMS
  2. WhatsApp Freelance Scam
  3. Spoofed Payment Gateway
  4. Fake Scholarship Portal
  5. Advance Fee Fraud
- Each module has a flip card (front: realistic scam scenario, back: red flags + verdict)
- After each card, a 3-question multiple choice quiz with instant feedback
- Streak counter, score tracker, and a completion badge

### API Endpoints to Build

```
POST /api/v1/analyze/content
Body: { content: string, contentType: "sms|whatsapp|url|email|other", sessionId?: string }
Response: { status, result: { classification, confidenceScore, threatType, redFlags[], explanation, recommendedAction }, processingMs }

POST /api/v1/banking/validate-transfer  
Body: { recipientName: string, iban: string, amountPKR: number, transferNote?: string, sessionId?: string }
Response: { status, result: { ibanValid, bankResolved, riskLevel, riskScore, behavioralFlags[], aiSummary, proceedRecommendation } }
```

### Backend System Prompt for AI (use this exactly)
```
You are ShieldPay's Financial Threat Intelligence Engine trained on Pakistan-specific banking fraud patterns. Analyze digital content for scam indicators.

CRITICAL RULES:
1. Return ONLY a valid JSON object. No markdown, no backticks, no preamble.
2. classification MUST be exactly one of: SAFE, SUSPICIOUS, DANGEROUS
3. confidenceScore MUST be an integer between 0 and 100
4. redFlags MUST be an array of strings. Empty array if none found.

OUTPUT SCHEMA: { classification, confidenceScore, threatType, redFlags[], explanation, recommendedAction }
```

### Typography
- **Display/Headers:** `font-family: 'Inter', sans-serif` — weights 700/800
- **Body:** `font-family: 'Inter', sans-serif` — weights 400/500
- **Code/IBAN display:** `font-family: 'JetBrains Mono', monospace`
- Import both from Google Fonts

### Component Design Rules
1. **Navbar:** Fixed top, height 64px, navy background, ShieldPay shield logo (SVG inline) on left, "UBL Hackathon 2026" badge on right
2. **Tabs:** Pill-style tabs, not underline — active tab has navy background with white text
3. **Result Cards:** Left 4px color accent bar (semantic color), white card body, soft shadow `shadow-md`, rounded-xl corners
4. **Buttons:** Primary = navy bg, white text, rounded-lg, hover darkens. Danger = crimson. Success = emerald.
5. **Input fields:** Rounded-lg, border-gray-200, focus ring in navy, comfortable padding
6. **Loading state:** Animated skeleton loader — not a spinner
7. **Confidence meter:** Horizontal progress bar, color matches classification state
8. **Flip cards:** CSS 3D transform perspective flip on click, not hover

## EDUCATION HUB CONTENT — BUILD THESE 5 MODULES

### Module 1: Fake Prize SMS
- **Scenario (Card Front):** You receive: "Congratulations! You've won a Samsung Galaxy S24 in UBL's 25th Anniversary draw! Your ticket #UBL2025-7742 was selected. Claim your prize: bit.ly/ubl-win-2025 — Offer expires in 24 hours!"
- **Red Flags (Card Back):** Urgency/time pressure, URL is not ubl.com, UBL doesn't run prize draws via SMS, no personalization
- **Quiz Q1:** What is the biggest red flag? A) The URL B) The prize value C) The company name ✓A
- **Quiz Q2:** What should you do? A) Click the link B) Call UBL official number C) Forward to friends ✓B  
- **Quiz Q3:** Legitimate banks contact you via? A) Random SMS links B) Official app notifications C) WhatsApp ✓B

### Module 2: WhatsApp Freelance Scam
- **Scenario:** Client DMs you on WhatsApp: "I need a logo ASAP. I'll pay Rs. 8,000 advance via JazzCash right now. Just send your number and full name. I'm in Dubai so can't do a call."
- **Red Flags:** Advance payment pressure, no formal brief, requesting personal info upfront, can't do a call (can't be verified)
- **Quiz:** 3 questions about verifying clients, platform-based payments, red flags in freelancing

### Module 3: Spoofed Payment Gateway  
- **Scenario:** You're buying concert tickets online. The checkout page looks exactly like JazzCash. It asks for your card number, expiry, CVV, and your JazzCash PIN. The URL is: jazzc4sh-payments.pk
- **Red Flags:** URL has '4' not 'a', no HTTPS lock, asking for PIN (real gateways never do), pressure to complete
- **Quiz:** 3 questions about URL inspection, CVV safety, and what real payment pages ask for

### Module 4: Fake Scholarship Portal
- **Scenario:** You see a Facebook ad: "HEC & UBL Joint Scholarship 2025 — Rs. 50,000 for students! Apply now at hec-ubl-scholarship.com — Application fee: Rs. 500 via EasyPaisa"
- **Red Flags:** Application fees are always fake, domain is not hec.gov.pk, advertised on Facebook not official channels, vague eligibility
- **Quiz:** 3 questions about verifying scholarships, official HEC portals, red flags in scholarship ads

### Module 5: Advance Fee Fraud
- **Scenario:** WhatsApp message: "Brother, I'm selling my Honda Civic 2019 for only Rs. 18 lakh, going to Canada urgently. Send Rs. 50,000 advance to hold the car via IBFT to IBAN: PK36MEZN... I'll deliver tomorrow."
- **Red Flags:** Below-market price to create urgency, IBFT advance for an unverified deal, won't meet in person, pressure framing
- **Quiz:** 3 questions about in-person verification, advance payments, and too-good-to-be-true deals
